# 量化交易项目方案调研

## 一、项目概述

### 1.1 什么是量化交易
量化交易是指利用数学模型、统计方法和计算机程序来进行交易决策的方式。
核心理念：用数据和规则替代人为主观判断。

### 1.2 量化交易的优势
- 消除情绪干扰（恐惧、贪婪）
- 纪律性执行策略
- 回测验证策略有效性
- 可同时监控多个标的
- 7x24小时运行（自动化）

## 二、技术栈选型

### 2.1 编程语言：Python（首选）
**原因：**
- 生态丰富（pandas, numpy, matplotlib）
- 量化库成熟（backtrader, vnpy, zipline）
- 社区活跃，学习资源多
- 开发效率高

### 2.2 核心技术栈

| 组件 | 推荐方案 | 备选方案 |
|------|----------|----------|
| 数据源 | Tushare Pro / AKShare | 聚宽、米筐、Wind |
| 回测框架 | Backtrader | vnpy, zipline, qlib |
| 实盘交易 | QMT（迅投）/ PTrade | easytrader（模拟） |
| 数据库 | SQLite（入门）/ MySQL | PostgreSQL, InfluxDB |
| 可视化 | Matplotlib + Plotly | pyecharts |
| 任务调度 | APScheduler | Celery, Airflow |

### 3.1 数据层（Data Layer）
**职责：** 获取、存储、管理各类市场数据

```python
# 数据类型
- 行情数据：日K、分钟K、Tick数据
- 财务数据：财报、估值指标
- 另类数据：舆情、资金流、龙虎榜
```

**推荐方案：Tushare Pro**
- 免费额度：每天500积分（约5万行数据）
- 覆盖：A股全市场行情 + 财务数据
- 接口：RESTful API + Python SDK

### 3.2 策略层（Strategy Layer）
**职责：** 开发、回测、优化交易策略

**常见策略类型：**

| 策略类型 | 说明 | 难度 | 适合入门 |
|----------|------|------|----------|
| 均线策略 | 金叉死叉 | ⭐ | ✅ |
| 动量策略 | 追涨杀跌 | ⭐⭐ | ✅ |
| 价值投资 | 低估值买入 | ⭐⭐ | ✅ |
| 配对交易 | 统计套利 | ⭐⭐⭐ | ❌ |
| 机器学习 | 预测涨跌 | ⭐⭐⭐⭐ | ❌ |
| 高频做市 | 流动性提供 | ⭐⭐⭐⭐⭐ | ❌ |

### 3.3 交易层（Trading Layer）
**职责：** 执行交易信号，管理订单和仓位

**A股实盘方案：**

| 方案 | 说明 | 门槛 |
|------|------|------|
| QMT（迅投）| 券商官方量化接口 | 需开通权限，资金门槛低 |
| PTrade | 恒生电子量化平台 | 部分券商支持 |
| easytrader | 模拟键鼠操作 | 无门槛，但不稳定 |

### 3.4 监控层（Monitoring Layer）
**职责：** 监控策略运行状态，异常报警

**功能：**
- 实时盈亏监控
- 策略异常报警（邮件/微信）
- 交易日志记录
- 绩效统计报表

## 四、入门路线图

### Phase 1：基础学习（1-2周）

```
├── 经典策略复现
│   ├── 双均线策略

├── 仓位管理策略
└── 策略组合
```


### Phase 4：实盘上线（持续）

```
Week 11+:
├── 开通券商量化权限
```

## 六、风险提示

### 6.1 量化交易的风险
- **过拟合风险**：回测很好，实盘亏损
- **市场风险**：黑天鹅事件、政策变化
- **技术风险**：程序bug、网络中断
- **流动性风险**：无法及时成交

### 6.2 入门建议
1. 先用模拟盘验证，不要直接实盘
2. 初期投入小资金（如1-2万）测试
3. 不要追求高收益，先追求稳定
4. 保持学习，市场在变化
